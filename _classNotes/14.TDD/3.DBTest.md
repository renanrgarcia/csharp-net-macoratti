# DB Test
- **Database Testing**: A type of testing that focuses on validating the interactions between an application and its database. This includes testing CRUD operations, data integrity, and performance.

## Concepts
- **Test Database**: A separate database used exclusively for testing purposes. It should be isolated from the production database to prevent data corruption.
- **Test Data**: The data that is used to populate the test database. It should be representative of real-world data and cover various edge cases.
- **Database Seeding**: The process of populating the test database with initial data before running tests.
- **Database Cleanup**: The process of removing test data from the database after tests have been executed to maintain a clean state for subsequent tests.

## Usage
```csharp
using Xunit;

public class UserServiceTests
{
    [Fact]
    public void GetUser_ShouldReturnUser_WhenUserExists()
    {
        // Arrange
        var fakeRepo = new FakeUserRepository();
        var userService = new UserService(fakeRepo);
        var user = new User { Id = 1, Name = "Test User" };
        fakeRepo.AddUser(user);

        // Act
        var result = userService.GetUser(1);

        // Assert
        Assert.Equal(user, result);
    }
}
```

```csharp
using Xunit;

public class UserServiceTests
{
    [Fact]
    public void CreateUser_ShouldAddUserToDatabase()
    {
        // Arrange
        var testDb = new TestDatabase();
        var userService = new UserService(testDb);
        var newUser = new User { Name = "New User" };

        // Act
        userService.CreateUser(newUser);

        // Assert
        var createdUser = testDb.GetUserByName("New User");
        Assert.NotNull(createdUser);
        Assert.Equal("New User", createdUser.Name);
    }
}
```
