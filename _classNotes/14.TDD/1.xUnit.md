# xUnit
- xUnit is a free, open-source, community-focused unit testing tool for the .NET Framework.

## Concepts
- **Fact**: An attribute used to denote a test method in xUnit.
- **Theory**: An attribute used to denote a parameterized test method in xUnit.
- **InlineData**: An attribute used to provide data for a parameterized test method.
- **Fixture**: A way to share setup and cleanup code across multiple test methods.

- **Arrange**: The first phase of the Arrange-Act-Assert (AAA) pattern, where you set up the necessary preconditions and inputs for the test.
- **Act**: The second phase of the AAA pattern, where you execute the code under test.
- **Assert**: The final phase of the AAA pattern, where you verify the outcome of the test.

## Usage

```csharp
using Xunit;

public class CalculatorTests
{
    [Fact]
    public void Add_ReturnsSum_WhenCalledWithTwoNumbers()
    {
        // Arrange
        var calculator = new Calculator();

        // Act
        var result = calculator.Add(2, 3);

        // Assert
        Assert.Equal(5, result);
    }
}
```

## Fact vs Theory
- **Fact**: A test method that does not take any parameters and is executed once.
- **Theory**: A test method that takes parameters and can be executed multiple times with different sets of data.

```csharp
using Xunit;

public class CalculatorTests
{
    [Theory]
    [InlineData(2, 3, 5)] // Test case 1: 2 + 3 = 5
    [InlineData(-1, 1, 0)] // Test case 2: -1 + 1 = 0
    [InlineData(0, 0, 0)] // Test case 3: 0 + 0 = 0
    public void Add_ReturnsSum_WhenCalledWithTwoNumbers(int a, int b, int expected)
    {
        // Arrange
        var calculator = new Calculator();

        // Act
        var result = calculator.Add(a, b);

        // Assert
        Assert.Equal(expected, result);
    }
}
```

- **Member Data**: A way to provide data for a parameterized test method from a static property or method within the same class.

```csharp
using Xunit;

public class CalculatorTests
{
    [Theory]
    [MemberData(nameof(AddTestCases))]
    public void Add_ReturnsSum_WhenCalledWithTwoNumbers(int a, int b, int expected)
    {
        // Arrange
        var calculator = new Calculator();

        // Act
        var result = calculator.Add(a, b);

        // Assert
        Assert.Equal(expected, result);
    }

    public static IEnumerable<object[]> AddTestCases =>
        new List<object[]>
        {
            new object[] { 2, 3, 5 },    // Test case 1: 2 + 3 = 5
            new object[] { -1, 1, 0 },   // Test case 2: -1 + 1 = 0
            new object[] { 0, 0, 0 },    // Test case 3: 0 + 0 = 0
        };
}
```

- **Class Data**: A way to provide data for a parameterized test method from a separate class.

```csharp
using Xunit;

public class CalculatorTests
{
    [Theory]
    [ClassData(typeof(AddTestCases))]
    public void Add_ReturnsSum_WhenCalledWithTwoNumbers(int a, int b, int expected)
    {
        // Arrange
        var calculator = new Calculator();

        // Act
        var result = calculator.Add(a, b);

        // Assert
        Assert.Equal(expected, result);
    }
}

public class AddTestCases : IEnumerable<object[]>
{
    public IEnumerator<object[]> GetEnumerator()
    {
        yield return new object[] { 2, 3, 5 };    // Test case 1: 2 + 3 = 5
        yield return new object[] { -1, 1, 0 };   // Test case 2: -1 + 1 = 0
        yield return new object[] { 0, 0, 0 };    // Test case 3: 0 + 0 = 0
    }

    IEnumerator IEnumerable.GetEnumerator() => GetEnumerator();
}
```
